# 0.1.4 — 2026-05-28

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.9` → `^0.9.13` — picks up HTTP transport hardening, log-level split for expected client errors, and GET /mcp keyword surfacing.
- **`fcc_compare_areas` error code** `invalid_all_states_combo` — reclassified from `InvalidParams` to `ValidationError`. Client-observable change.
- **`fcc_get_coverage_summary` error code** `invalid_geography_combo` — reclassified from `InvalidParams` to `ValidationError`. Client-observable change.
- **`fcc_list_downloads` error code** `invalid_as_of_date` — reclassified from `InvalidParams` to `ValidationError`. Client-observable change.
- **`geography-summary` resource** `invalidParams` factory → `validationError` factory for invalid geography type. Client-observable change.
- **`bdc-api-service`** `invalidParams` factory → `validationError` factory for malformed `as_of_date` format.
- **`landing.requireAuth: false`** set in `createApp()` — ensures GET /mcp serves the full tool/resource inventory on this public server now that 0.9.13 gates it by default when auth mode is non-none.
- **`@biomejs/biome`** `^2.4.15` → `^2.4.16`.
- **Keywords** `typescript`, `bun`, `stdio`, `streamable-http` added to `package.json` — now surfaced on GET /mcp status response.
